Median selection networks for a given number of inputs

The networks below can be used to find the median value of the inputs and are not full sorting networks. The rows that yield the output(s) are marked in blue. For an even number of inputs, the average of the two outputs still needs to be computed by the application. Note that in this case, the two outputs may not be ordered.

As for the sorting networks list, the median selection networks have been defined in terms of two metrics:

  • Number of compare/exchange elements (CEs) - metric below called "size"
  • Number of parallel operation steps (layers), determining processing latency - metric below called "depth"

Provided is a minimum set of networks that each can be the best choice depending on how both criteria are weighted.

If no source is mentioned, the network was generated by the SorterHunter program, by applying postfix networks to the evolving network and removing them from the resulting sorting network.
